\subsection{Plug and Play Language Models}
We build off the plug and play language model (PPLM) framework presented in \cite{Dathathri2020Plug}, which frames the controlled generation problem as generating from a distribution $p(x|a)$ where $a$ is some attribute (e.g. positivity). To do this in a lightweight manner, PPLM combines an attribute model, $p(a|x)$, and language model, $p(x)$, exploiting the relation: 

\begin{equation}
    p(x|a) \propto p(a|x)p(x)
\end{equation}

The attribute model $p(a|x)$ yields a scalar probability whereas the language model yields a probability distribution over the vocabulary, so combining these two models is not so straightforward in practice. In practice, the PPLM algorithm uses the language model to estimate $p(x|c)$, where $c$ is some prior context. During decoding, the attribute model computes $p(a|c)$, and the gradient signal from that calculation is used to adjust the language model's hidden state during decoding. PPLM then uses post-norm shallow fusion, i.e. adding the logits from this adjusted decoding process to those from an unaltered decoding process. This shallow fusion approach has a history in machine translation approaches of combining distributions from two distinct language models \citep{DBLP:journals/corr/GulcehreFXCBLBS15, gulcehre2017integrating}.

\subsection{Reinforcement Learning}
We fine-tune a language model using reinforcement learning, where a linear combination of the output from attribute discriminators is a reward. In training, we use the TRL library's implementation of the proximal policy optimization (PPO) algorithm for fine-tuning language models. Due to computational constraints, we also use the parameter efficient fine tuning (PEFT, \citet{peft}) library's implementation of low rank adapters (LoRA, \citet{hu2022lora}) to fine tune LLaMA.

We combine the attribute discriminators $d_1, d_2$ into a single reward function for generated text $x$, where specifically $R = w_1d_1(x') + w_2d_2(x')$. 
The scalar weights $w_1, w_2$ are $\frac{1}{n}$ or $0$ if $d$ is positive $d(x') < d(x)$. The intuition is that if the model is not improving on the baseline generation, we do not want to learn an associated reward.
